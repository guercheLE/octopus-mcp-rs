# Monitoring & diagnostics

This is a thin pointer, not a multi-step flow: for "what's the state of X" questions, search for the read-only operation that answers it directly rather than following a guided procedure — the dashboard (overall deployment/runbook status across projects), tasks (a specific deployment/runbook-run's status and log — poll status before fetching a full log, and delegate the log fetch to an isolated sub-task if your environment supports one and the log is long), events (the audit log), deployment/runbook progression (per-environment history for a project), interruptions (pending manual-intervention/approval steps awaiting attention — this prompt is for *listing* them; to actually claim and resolve one, use `octopus_workflow_manual_intervention` instead), and server status (overall Octopus Server health). Read the schema `get` returns for whichever operation you pick before relying on any field name, and prefer a scoped/filtered query (by project, environment, or time range) over an unfiltered listing where the operation supports one, since several of these can return large result sets.
